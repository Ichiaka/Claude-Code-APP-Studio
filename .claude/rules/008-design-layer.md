# Reglas para documentos de diseño

- **PRDs**: deben tener problema, usuario, criterios de aceptación, métrica de éxito.
- **Feature specs**: deben cubrir estados (vacío/cargando/error/éxito) y edge cases.
- **Decisiones de diseño**: si afectan a arquitectura, también necesitan ADR.
